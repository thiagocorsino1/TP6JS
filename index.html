2)¿Cómo se integra JavaScript a HTML?.

Javascript se puede integrar de 3 maneras a html:

  • Con la etiqueta Script: se agrega la etiqueta script en el body y dentro de ella las sentencias.
  • En un Archivo Externo: se enlaza un archivo.js al html por un src.
  • En Linea: se agregan las sentencias dentro de una etiqueta html.

3)¿Cómo es el manejo de las variables, tipos de datos y expresiones regulares en JS?.

Variables

las variables se manejan con nombres simbólicos para darles valores en la aplicación. Los nombres de las variables, llamados identificadores, se ajustan a ciertas reglas. Un identificador de JavaScript debe comenzar con una letra, un guión bajo ( _ ) o un signo de dólar ( $ ). Las variables se pueden declarar usando var , const , o let.

Tipos de Datos

Se manejan los siguientes tipos de datos en JS

  • String:Cadenas de texto
  • Number:Valores numéricos.
  • Bigint:tipo de dato numerico que puede representar números enteros en el formato de precision arbitrario
  • Boolean:Representa una entidad lógica y puede tener dos valores: true y false.
  • null:Es un valor asignado tiene el valor de “no valor”.
  • undefined:Una variable a la que no se le ha asignado ningún valor tiene el valor undefined.
  • Symbol:Son siempre valores distintos aunque tengan el mismo nombre.
  • Object:Un valor en memoria al que podemos acceder por un identificador.

Expresiones Regulares

Las expresiones regulares se utilizan con los métodos de RegExp test y exec y con los metodos de String match , replace , search , and split.

4)¿Cómo son todas las formas de declaración de funciones en JS?.

Function declaration

Este tipo de función se creará con la palabra reservada function , seguido obligatoriamente por un nombre, que identificará a nuestra función, una lista de parámetros entre paréntesis, y el símbolo de las llaves {}. Qué será el que delimitará el contenido de nuestro conjunto de sentencias. La lista de parámetros cuando creamos una función no es obligatoria, podemos definir una función sin parámetros.

Function expression

Esta forma de declarar una nueva función es muy similar a la forma de declaracion de anterior, "Declaración de función". La sintaxis para crear estas funciones es similar a la anterior. La única diferencia es que la definición de nuestra nueva función no comienza por instrucción function y el nombre de la función es opcional y, este tipo de funciones las podemos almacenar en una variable.

IIFE (Immediately Invoked Function Expression)

Normalmente cuando definimos una función es porque la vamos a llamar en varias ocasiones, pero sí sólo queremos llamar a la función en una sola ocasión y obtener un resultado. Para eso podemos usar las funciones IIFE. Este tipo de funciones se ejecutan inmediatamente y no son accesibles posteriormente. Para crear una función de este tipo debemos crearla en un operador de agrupación (). Seguido de (), lo que permitirá el interpretado directamente en el motor de JavaScript.

Shorthand method definition

Para crear este tipo de función debemos asignar un nombre de función seguido de una lista de parámetros entre paréntesis y los símbolos de llaves para delimitar el cuerpo de las instrucciones.

Arrow function

La forma de crear estas funciones es la siguiente: Primero definiremos la lista de parámetros, en caso de ser necesario, entre paréntesis seguido del símbolo => y las {} para indicar las instrucciones que se van a realizar.

Generator function

La característica más significativa de estas funciones es que nos permite parar o salir de la función en un punto dentro del conjunto de instrucciones que forman nuestra función y retornar posteriormente la ejecución desde el punto en el que paramos anteriormente.

Function constructor

Con esta funcion podremos crear una variable que invoque a un objeto Function. Cuando invoquemos a este objeto podremos enviar todos los argumentos que deseemos. Los primeros n argumentos serán los parámetros de nuestra función y el último argumento será el código de nuestra función.

5)¿Cuáles son los operadores y conversión entre tipos?.

En JavaScript encontramos cinco tipos de operadores:

  • Aritméticos

    • suma(+), resta(-), negativo(-), division(/), modulo aritmetico %, incrementos y decrementos(++ --).

  • Lógicos

    • Negación lógica ! (Not). Establece una negación lógica en una expresión, es decir, que ante una expresión, cuyo estado lógico es True (verdadero), el operador hará que devuelva False (falso). Conjunción lógica && (And). Establece una conjunción lógica de dos expresiones, es decir, que han de resultar True (verdadero) las dos expresiones para que el resultado final también lo sea. Disyunción lógica || (Or). Establece una disyunción lógica de dos expresiones, es decir, que el resultado se dará evaluando una expresión u otra. Exclusión lógica ^ (Xor). Establece una exclusión lógica de dos expresiones, es decir, que el resultado se dará evaluando una expresión u otra, y dará True si solamente una de ellas es True, lo que implica la exclusión de la otra.

  • De comparación
    • Mayor que (>)
    • Mayor o igual que (>=)
    • Menor que ( <)
    • Menor o igual que ( <=)
    • igualdad (==) Verifica la igualdad de dos expresiones sin tener en cuenta el tipo de dato
    • Igualdad estricta (===) Hace lo mismo que el anterior, pero verificando también que coincidan los tipos de datos
    • Desigualdad (!=) Funciona de la misma forma que la igualdad, pero negándola, Desigualdad estricta (!==) Lo mismo que la igualdad estricta, pero negándola
  • Binarios

    • Existen algunos operadores que funcionan bit a bit, convirtiendo previamente los valores a binario.
    Son los siguientes: <<,>>, >>>, &, |, ^.

  • De asignación

    • Asignación simple (=), Asigna un contenido a una variable o a un objeto. Por ejemplo: fruta = "banana".

6)¿Cuáles son las estructura de control de JS?.

‌Las estructuras de control de flujo, son intrucciones que nos permiten evaluar si se puede cumplir una condición o no, incluso nos pueden ayudar a evaluarla n cantidad de veces.

tipos de estructuras de control:

  • condicionales

    • Los condicionales no permiten evaluar si una condición cumple o no con lo que estamos evaluando. Su sintaxis es muy sencilla, podemos evaluar si la condición es verdadera o falsa. Incluso agregue una condición intermedia en el caso de que no se cumpla la primera condición y se deban evaluar más. En estos podemos encontrar el if y el if-else.

  • Ciclos, Bucles o Loops

    • Se le pueden llamar, ciclos, bucles o loops, en ellos se evalúa una condición n veces hasta que esta se cumpla. En estos podemos encontrar los for, for in, while, swich.‌

7)¿Qué es el objeto dom en JS? Y ¿Cómo trabaja?.

Las siglas DOM significan Document Object Model, o lo que es lo mismo, la estructura del documento HTML. Una página HTML está formada por múltiples etiquetas HTML, anidadas una dentro de otra, formando un árbol de etiquetas relacionadas entre sí.
En Javascript,cuando nos referimos al DOM nos referimos a esta estructura, que podemos modificar de forma dinámica desde Javascript, añadiendo nuevas etiquetas, modificando o eliminando otras, cambiando sus atributos HTML, añadiendo clases, cambiando el contenido de texto, etc.

8)¿Qué función tiene en JS el Object Map?.

El método map() crea un nuevo array con los resultados de la llamada a la función indicada aplicados a cada uno de sus elementos, permite iterar y controlar la forma en que procesaremos cada unos de los elementos del arreglo en un solo paso.
